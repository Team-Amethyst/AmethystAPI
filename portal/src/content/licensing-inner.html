Licensing

Amethyst Engine is a generalized Sports Data as a Service (SDaaS) platform. The core API is available for commercial licensing under a revenue-share model. Licensee-specific data instances and strategic configurations remain private and isolated.

API access & keys

Licensed Engine routes require an x-api-key header on every request. Keys are issued through this portal, stored hashed in our database, and tied to your developer account for usage reporting.

  • Sign in, then mint keys — After sign in, use the API keys tab to create and revoke keys for your developer account. The full secret is shown once at creation; only a prefix remains in the list afterward.
  • Tier & scope — Each key has a tier (free, standard, or premium) that controls rate limits, plus scopes that match which Engine surfaces the key may call (valuation, catalog, scarcity, simulation, signals).
  • Usage — Requests are counted per key for royalty reporting and the usage tools on the API keys page (you can paste any key there to refresh counts without signing in).

Operators may turn issuance off or require an operator token for scripted POST /api/keys/issue flows; the API keys tab reflects live issuance status from the API.

Revenue Share Model

Commercial licenses are structured as a percentage of net revenue generated by your application through use of the Amethyst Engine API. This aligns our incentives with your product's success.

Model

Term
5 yr

Under the Reuse Clause of the Amethyst Engine license agreement, founding customers receive a 5% royalty on net licensing revenue for a period of five years following the first commercial sale. This royalty is calculated and reported quarterly based on cumulative API key usage tracked per licensee developer account (each issued key contributes to reporting).
